# SuperFolder 文件浏览器设计

## 定位

SuperFolder 是运行在 App Host 之上的高性能文件管理工作台。它的主要目标是快速浏览和操作文件夹，并逐步扩展收藏目录、自定义右键菜单、Git/P4 信息、Terminal、文件预览和多窗口工作流。

前端负责渲染和运行期交互状态；文件系统访问、目录数据、配置、会话恢复、Git/P4、Terminal 和预览等能力由后端通过 RPC 提供。

## 核心目标

- 面向 Windows 桌面体验。
- 支持 Go 本地入口加载 Web UI，也支持普通浏览器访问同一套能力。
- 前端使用 TypeScript/React/Vite，后端文件浏览能力使用 Go 实现。
- 支持多个 OS 级 Workspace Window。
- 每个 Workspace Window 支持独立布局、文件浏览 pane、tab 和底部多功能面板。
- 默认主界面左侧是收藏/文件夹树，主区域上方默认左右两个文件浏览 pane，下方是可折叠、可调整高度的多功能面板。
- 文件浏览需要高响应、低 CPU、低内存占用。
- 所有持久化状态由后端保存，前端只保存运行期可见状态。

## 主界面结构

界面分为三层：

1. 左侧 Sidebar：收藏目录树，也可以展示用户固定的工作区、项目和常用目录。
2. 上方 Main Browser Area：默认左右并列两个 File Browser Pane。每个 pane 可以有多个 tab。
3. 下方 Utility Panel：可折叠、可调整高度的大面板，包含 Terminal、P4、Git、文件预览等大 tab。

默认布局是左右双栏，但底层采用通用 Split Pane 模型。未来可以支持更多 pane、横向/纵向拆分、布局保存和恢复。

## 关键术语

- Workspace Window：一个 OS 级 SuperFolder 窗口，由同一个 App Host 管理。
- Split Pane：通用布局容器，支持多个文件浏览区域组合。
- File Browser Pane：独立文件浏览区域，拥有自己的 tab、当前路径、视图模式、排序/过滤和数据订阅。
- Browser Tab：File Browser Pane 内的一个浏览上下文，保存当前路径、历史、视图设置和打开状态。
- Favorites Tree：左侧收藏目录树，以树形结构组织用户收藏的文件夹。
- Utility Panel：底部多功能面板，承载 Terminal、Git、P4、Preview 等功能。
- View State：前端内存中的临时交互状态，例如滚动位置、可见范围、hover、focus。
- Session Persistence：后端保存的可恢复工作状态，例如打开的 tab、当前路径、窗口布局和视图偏好。

## 多窗口模型

同一个 App Host session 可以管理多个 Workspace Window。每个窗口独立保存：

- 窗口尺寸和位置。
- Split Pane 布局。
- File Browser Pane 列表。
- 每个 pane 的 tabs。
- 底部 Utility Panel 状态。
- 当前激活 pane、tab 和主要视图偏好。

多个 Workspace Window 不应该启动多个独立 App Host。这样可以共享缓存、目录 watch、Git/P4 状态和配置存储，减少内存和 CPU 占用。

## 文件浏览模型

前端维护当前可见目录树和列表的运行期状态。每次目录展开、路径切换、排序/过滤变化、视图模式变化，都通过 RPC 请求后端。

后端负责：

- 读取目录。
- 维护目录缓存。
- 合并文件系统 watch 事件。
- 返回目录快照和增量变化。
- 按请求范围返回可见数据。
- 计算或合并 Git/P4 状态。
- 处理大目录、慢磁盘和网络路径的取消与超时。

前端负责：

- 渲染当前可见数据。
- 保存滚动位置、可见范围、hover、focus 等内存状态。
- 发出展开、切换、排序、过滤、打开文件、右键菜单等用户意图。
- 在收到后端 push 后更新可见区域。

打开的文件、打开的目录 tab、当前路径和窗口布局需要由后端恢复。滚动位置这类交互状态可以重启后重置。

## 视图模式

File Browser Pane 应支持多种浏览方式，第一阶段可以设计为：

- 详情列表：适合大量文件、排序和状态列。
- 平铺/图标：适合图片、资源和视觉浏览。
- 树状视图：适合在一个 pane 内展开目录层级。

视图模式只改变前端呈现和后端请求的范围/字段，不应改变目录能力模型。

## 收藏目录树

左侧 Favorites Tree 是持久化配置，由后端保存。它以树形结构展示收藏目录，可以支持：

- 固定目录。
- 分组。
- 重命名显示名。
- 排序。
- 展开/折叠状态。
- 指向本地路径、网络路径或仓库根目录。

收藏树不是文件系统真实树的完整镜像，而是用户组织工作入口的配置树。

## 自定义右键菜单

右键菜单由后端保存配置并负责执行能力。前端只负责根据上下文请求可用菜单项并展示。

菜单项需要能够感知：

- 当前选中文件/目录。
- 当前 pane/tab。
- 路径类型。
- Git/P4 状态。
- 是否多选。
- 当前应用能力和权限。

执行菜单项时应通过 RPC 发起 action，后端返回进度、结果或错误。高风险操作需要确认机制。

## Git 与 P4 信息

Git/P4 状态不应由前端扫描文件系统。后端负责识别仓库、缓存状态、读取 log，并把变化推送给订阅中的窗口/pane/tab。

第一阶段至少需要定义：

- 文件列表中的状态字段。
- Git/P4 log 的 Utility Panel tab。
- 后端状态刷新和取消机制。
- 大仓库或慢命令的进度反馈。

Git/P4 功能应作为 SuperFolder 后端能力模块接入 RPC，不应写入通用 App Host 框架。

## Utility Panel

底部 Utility Panel 是一个可折叠、可调整高度的多功能区域。它有大 tab，每个大 tab 后续可以单独设计。

初始大 tab 包括：

- Terminal
- P4
- Git
- Preview

Utility Panel 的高度、折叠状态、激活 tab 应由后端保存。具体 tab 内容可以逐步设计，不要求第一版一次完成全部深度功能。

## 持久化边界

后端保存：

- 收藏目录树。
- 自定义右键菜单。
- Workspace Window 布局。
- Split Pane 布局。
- 打开的 File Browser Pane 和 Browser Tab。
- 当前路径。
- 视图模式、排序、过滤等偏好。
- Utility Panel 状态。
- Git/P4 配置。
- 文件打开状态和会话恢复数据。

前端内存保存：

- 滚动位置。
- 可见范围。
- hover/focus。
- 临时拖拽状态。
- 临时选择框和局部交互状态。

## 性能原则

- 文件列表使用虚拟化渲染。
- 大目录按可见范围和订阅状态加载。
- 后端目录读取、Git/P4 状态、预览生成需要可取消。
- 文件系统变化通过后端 push 增量更新，不依赖前端轮询。
- 多窗口、多 pane、多 tab 共享后端缓存和 watch。
- 前端不做持久化数据库，不重复维护大目录全集。
- 右键菜单、状态列和预览信息需要懒加载。

## 当前决策

- SuperFolder 使用 App Host/RPC 提供能力。
- 本地入口为 Go `.exe` + Web UI。
- Web 和本地入口前端功能一致，差异主要是启动方式、渲染容器和本地效率。
- 前端使用 TypeScript/React/Vite，后端使用 Go。
- 默认主界面是左侧收藏树、上方左右两个文件浏览 pane、下方 Utility Panel。
- 底层布局采用通用 Split Pane 模型，默认左右双栏。
- 支持多个 OS 级 Workspace Window，由同一个 App Host 管理。
- 前端只保存运行期 View State；可恢复工作状态和配置由后端保存。
- RPC 支持双向调用和后端推送。

## 后续设计主题

- 第一版文件操作范围：浏览、打开、复制、移动、删除、重命名是否都纳入。
- 第一版右键菜单配置格式和安全限制。
- 文件预览第一版支持哪些类型。
- Terminal 是否第一版实现，还是只保留 Utility Panel 入口。
- Git/P4 第一版显示到什么深度：状态列、log、diff、history。
- 远程访问时的文件操作权限和认证模型。
